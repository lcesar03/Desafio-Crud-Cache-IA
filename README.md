🧩 Desafio: CRUD com Cache

Você deve desenvolver uma API REST simples que gerencie uma entidade (livre escolha, ex: usuário, produto, tarefa, etc).

🎯 Objetivo

Construir um CRUD completo com integração a banco NoSQL e uso de cache.

⸻

⚙️ Stack obrigatória
 • Spring Boot
 • MongoDB
 • Redis

⸻

📌 Requisitos

🔹 API REST (4 operações obrigatórias)

Sua aplicação deve expor endpoints para:
 • Criar um registro
 • Listar registros
 • Atualizar um registro
 • Deletar um registro

⸻

🔹 Persistência
 • Utilizar MongoDB como banco principal
 • Estruturar corretamente entidades e repositórios

⸻

🔹 Cache
 • Utilizar Redis como camada de cache
 • Pelo menos um endpoint deve utilizar cache
 • Deve haver estratégia de invalidação

⸻

🔹 Boas práticas esperadas
 • Organização em camadas (Controller, Service, Repository)
 • Uso adequado de anotações do Spring
 • Código limpo e legível

⸻

🧪 Diferenciais (opcional)
 • Paginação
 • Validação de dados
 • Documentação com Swagger/OpenAPI

⸻

🚀 Entrega esperada
 • Projeto funcional
 • Código versionado (Git)
 • Instruções para rodar o projeto
